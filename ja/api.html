REST ゲートウェイ¶

Catapult REST は HTTP と WebSockets をブロックチェーン上で読み書きアクションが実行できるように併合します。

HTTP リクエスト¶

REST ゲートウェイはポート 3000 番を使用します。HTTP GET 、 PUT そして POST リクエストを受け付けます。

Catapult REST がローカルで動いているのであれば、HTTP GET リクエストはブラウザからこのように実行できます:

http://localhost:3000/<path-to-API-request>

Conversely, HTTP PUT and POST requests have the same structure but use JSON structures in the request body. This kind of request cannot usually be executed from within the browser unless you use a plugin which enables you to do it.

次のドキュメントを参照して、使用可能なエンドポイントのリストを取得できます。

  • v0.8.12

WebSockets¶

ブロックチェーン上でのイベントの発生による リアルタイムな更新 を得るために、 Catapult REST は WebSocket を公開しています。クライアントアプリケーションは WebSocket 接続を開くと一意な識別子を得られます。この識別子によって、アプリケーションは更新のために API を絶えずポーリングするのではなく、使用可能なチャンネルを購読する資格を得ます。チャンネルでイベントが発生すると REST ゲートウェイは購読しているすべてのクライアントにリアルタイムで通知を送信します。

WebSocket URI は HTTP リクエスト URI と同じホストとポートを共有しますが ws:// プロトコルを使用します:

ws://localhost:3000/ws
  • ガイド: WebSocket チャンネルの購読

チャンネル¶

ブロック

ブロックチャンネルは、新しくハーベストされたブロックがあるたびに、購読しているすべてのクライアントに通知します。返されるメッセージにはブロックに関する情報が含まれています

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "block"
}

レスポンスフォーマット

  • BlockInfoDTO

confirmedAdded/{address}

承認済追加チャンネルは、アドレスに関連するトランザクションがブロックに含まれたときに通知します。返されるメッセージには、承認済みのトランザクションに関する情報が含まれています。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "confirmedAdded/{address}"
}

レスポンスフォーマット

  • TransactionInfoDTO

unconfirmedAdded/{address}

未承認追加チャンネルは、アドレスに関連するトランザクションが未承認状態にあり、ブロックに含まれるのを待っているときに通知します。返されるメッセージには、未承認のトランザクションに関する情報が含まれています。

トランザクションが PUT /transaction HTTP エンドポイントによりアナウンスされるか、アグリゲートボンドトランザクションのすべての署名が揃い、partial から未承認へ変わるとき、このチャンネルからの通知が届く可能性があります。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "unconfirmedAdded/{address}"
}

レスポンスフォーマット

  • TransactionInfoDTO

unconfirmedRemoved/{address}

未承認削除チャンネルはアドレスに関連するトランザクションが未承認状態であったが、そうでなくなったときに通知を行います。返されるメッセージには、トランザクションハッシュが含まれます。

このチャンネルが通知するシナリオとしては、トランザクションが承認済になった、または期限を迎えてブロックに含まれなくなったときです。

リクエストボディ

{
    "uid":"{uid}",
    "subscribe":"unconfirmedRemoved/{address}"
}

レスポンスフォーマット

  • Hash

partialAdded/{address}

パーシャル追加チャンネルはアドレスに関連するアグリゲートボンドトランザクションが partial な状態にあり、すべての必要な連署者を待機しているときに通知を行います。返されるメッセージには、トランザクションに関する情報が含まれています。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "partialAdded/{address}"
}

レスポンスフォーマット

  • TransactionInfoDTO

partialRemoved/{address}

パーシャル削除チャンネルはアドレスに関連するトランザクションがパーシャルな状態であったが、そうでなくなったときに通知を行います。返されるメッセージには、トランザクションハッシュが含まれます。

このチャンネルが通知するシナリオとしては、トランザクションが未承認になった、または期限を迎えてブロックに含まれなくなったときです。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "partialRemoved/{address}"
}

レスポンスフォーマット

  • Hash

cosignature/{address}

連署チャネルはアドレスに関連する 連署署名トランザクション が、パーシャル状態のアグリゲートボンドトランザクションに追加されたときに通知します。返されるメッセージには、署名済み署名付きトランザクションが含まれています。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "cosignature/{address}"
}

レスポンスフォーマット

  • CosignatureDTO

status/{address}

ステータスチャンネルはアドレスに関連するトランザクションでエラーが発生したときに通知を行います。返されるメッセージには、エラーメッセージとトランザクションハッシュが含まれます。

リクエストボディ

{
    "uid": "{uid}",
    "subscribe": "status/{address}"
}

レスポンスフォーマット

  • TransactionStatusDTO
