ワークステーションのセットアップ¶

最初のガイドでは Symbol 開発を始めるために必要なツールのインストールをステップバイステップに進めていきます。

は将来のパブリックメインネットワークと同じテクノロジーと機能を反映している テストネットワーク を使用します。テストネットを実験的に使用することで、貴重な資産を失うことなく Symbol が提供するトランザクションセットを試すことができます。

アカウントの作成¶

アカウント はあなたが モザイク (トークン) を保有することのできる、預け入れ箱であり、 トランザクションをアナウンス することでこれらと疎通できます。

注釈

Symbol CLI は Node.js 12 LTS を実行に必要とします。

  1. 次のコマンドを新しいターミナルで実行して Symbol CLI をインストールします:
npm install --global symbol-cli@0.19.2
  1. 新しいアカウントを生成して profile として保存します
symbol-cli account generate

Enter network type (MIJIN_TEST, MIJIN, MAIN_NET, TEST_NET): TEST_NET
Do you want to save the account? [y/n]: y
Enter Symbol Node URL. (Example: http://localhost:3000): http://api-01.ap-northeast-1.testnet-0951-v1.symboldev.network:3000/
Insert the profile name: testnet
Do you want to set the account as the default profile? [y/n]: y

注釈

テストネットワークノードが動いていない場合は this list から他のノードを使用できます。この ガイド に従って テストネットノードを実行 することもできます。

  1. アカウントの資格情報がターミナルで確認できます。
Profile stored correctly

│ Property    │ Value                                                            │

注釈

プライベートキーは必ず秘密であることを守ってください。オフラインのどこか安全な場所に秘密鍵がバックアップされていることを確認してください。

テスト通貨の取得¶

トランザクションをアナウンスするには、ネットワークを保護・検証するインフラストラクチャを維持する人々へのインセンティブを提供するための 手数料 を支払う必要があります。このコストはデフォルトのパブリックネットワークのネットワーク通貨である symbol.xym モザイクで支払われます。

最初のアカウントを作成したので、 テストネットフォーセット から|networkcurrency| の取得をリクエストできます。 Navigate here で受信したい symbol.xym の量とアドレスを指定して 「CLAIM!」 をクリックします。

注釈

デフォルトフォーセットが枯渇している場合は alternative faucet を試してください。

トランザクションが承認されたら、アカウントが symbol.xym を受信していることをコマンドラインツールを使用して確認します。

symbol-cli account info --profile testnet

Balance Information

├──────────────────┼─────────────────┼─────────────────┼───────────────────┤
│ 05D6A80DE3C9ADCA │ 750.0           │ 750000000       | Never             │

The faucet has a limited amount of symbol.xym and must be replenished before it dries. If you don’t need your test symbol.xym units anymore, please send them back to the account TA6W43-2Q3NHN-Q5HQSK-Q52XL7-MFJZQI-TJRYBI-LQGG.

プロジェクトの作成¶

アカウントに symbol.xym 単位が入力されたので プログラミング言語を選択 しましょう。あなたが一番使いやすいと思う、またはあなたのプロジェクトに必要な言語を選んでください。

次に、 新しいプロジェクト用のフォルダを作成します 。そして、選択した言語による指示に従います。プロジェクトで使用する言語がどれにも当てはまらない場合は REST ゲートウェイ を使用して直接ブロックチェーンに問い合わせできます。

TypeScript
JavaScript
Java

注釈

Symbol SDK は Node.js 12 LTS を実行に必要とします。

  1. package.json ファイルを作成します。
npm init
  1. Symbol SDK と RxJS ライブラリのインストール。
npm install symbol-sdk@0.19.2 rxjs
  1. TypeScript の依存をグローバルにインストールします:
npm install --global typescript
typescript --version
  1. ts-node をインストールすると TypeScript ファイルを node で実行できます。
sudo npm install --global ts-node

注釈

Symbol SDK requires Node.js 12 LTS to execute.

  1. package.json ファイルを作成します。
npm init
  1. Symbol SDK と RxJS ライブラリのインストール。
npm install symbol-sdk@0.19.2 rxjs

注釈

Symbol SDK requires JDK 8 to execute.

  1. 新しく Java gradle プロジェクトを開いてください。お好みの IDE を使用するか、コマンドラインからプロジェクトを作成してください。
gradle init --type java-application
  1. Maven セントラルリポジトリを使うため build.gradle を編集します。
repositories {
    mavenCentral()
}
  1. symbol-sdk を依存として追加します。
dependencies {
    compile "io.nem:symbol-sdk-vertx-client:0.19.0"
}
  1. プログラムを開始するために gradle build と gradle run を実行します。
