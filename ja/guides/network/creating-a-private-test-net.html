プライベートテストネットの作成¶

このガイドでは、 Symbol のテクノロジーを使用して、独自のプライベートネットワークをセットアップするプロセスを説明しています。

ネットワークを立ち上げるために Catapult Service Bootstrap を使用します。このソフトウェアスイートは開発者が素早く自分のネットワークを構築するための手助けになるセットアップスクリプトが含まれています。

注釈

このブートストラップの設定は学習と開発を目的としており、本番の Symbol インスタンスを動かすものではありません 。

Symbol Service Bootstrap は次の高レベルアーキテクチャでプライベートネットワークを展開しています:

ブートストラップネットワークアーキテクチャ

  • peer-node (1 と 2): Peer ノードは API がトランザクションを P2P ネットワークにプッシュすると、それを検証します。ノードはコンセンサスアルゴリズムを実行して、新しいブロックを作成して、ネットワークを通して変更を伝播します。
  • api-node: API ノードはトランザクションが検証された後に MongoDB データベースにデータを保存します。これらはパーシャルなアグリゲートボンドトランザクションを識別して保存もします。
  • rest-gateway は HTTP と WebSockets をブロックチェーン上で読み書きアクションが実行できるように構成します。

ハードウェア要求¶

Symbol Service Bootstrap は次の 最低必要環境 でテストされています。

  • CPU: 2 コア以上
  • Memory: 4GB 以上
  • HD: 20GB 以上

注釈

それほど強力ではないインスタンスでソフトウェアを実行できる場合もありますが、ノードのインストールまたは実行中に問題が発生する場合があります。

必須環境¶

Linux と Mac OS 用のセットアップスクリプトは Docker を使用して自動化されます。テストネットノードを立ち上げるには、次の Docker ツールをインストールする必要があります。

  • docker
  • docker-compose

次の必須項目¶

サーバーのホストがインターネットからアクセス可能であり、次のポートが開いていて、使用可能であることを確認してください。

  • 7900 番ポートはノード間の通信するに catapult-server が使用します。
  • 3000 番ポートは REST ゲートウェイがエンドポイントを公開してノードとの疎通に使用します。

インストール¶

  1. パッケージの 最新版のダウンロードリンク またはリポジトリから Git を使用して直接クローンしてください。
git clone https://github.com/tech-bureau/catapult-service-bootstrap.git
  1. catapult-service-bootstrap フォルダを開きます。
cd catapult-service-bootstrap
  1. (オプション) 起動の前に、ネットワークの 設定可能なパラメータ をカスタマイズします。
  2. ネットワークの立ち上げ
./cmds/start-all

注釈

ターミナルのバックグラウンドで Docker コンテナを実行するには --detach または -d オプション使用してサービスをデタッチモードで実行します。

  1. 次の URL で新しいブラウザタブを開いてノードが実行されていることを確認します: localhost:3000/chain/height

プロセスを停止するには Ctrl+C を押下します。

インストレーションノート でより多くのネットワークマネジメントコマンドを見ることができます。

テスト通貨の取得¶

アカウント はあなたが モザイク (トークン) を保有することのできる 預け入れ箱 であり、トランザクションをアナウンスすることでこれらと疎通できます。トランザクションをアナウンスするには、ネットワークを保護・検証するインフラストラクチャを維持する人々へのインセンティブを提供するための 手数料 を支払う必要があります。このコストはデフォルトのネットワークトークンである cat.currency モザイクで支払われます。

初回の catapult-service-bootstrap ツールの起動後、利用可能な通貨の供給は、生成されたアカウントに分配されます。これらのアカウントの 1 つをすばやく取得するには、ブロックチェーンとの対話、アカウントの設定、資金の送信など、最も一般的なアクションを便利に実行するコマンドラインツールを使用して保存します。

  1. Symbol CLI のインストール
npm install --global symbol-cli

2. Open a new terminal window. Then, go to the directory where the bootstrap tool has generated the addresses.

cd  build/generated-addresses/
  1. addresses.yaml ファイルの内容を表示します。
cat addresses.yaml

4. Under the section nemesis_addresses, you will find the key pairs which contain cat.currency. Copy the private key of the first account.

  1. 前のステップで取得したキーを使用して、コマンド symbol-cli profile import を実行します。
symbol-cli profile import

Enter network type (MIJIN_TEST, MIJIN, MAIN_NET, TEST_NET): TEST_NET
Enter your private key: 123***456
Enter a Symbol Node URL. (Example: http://localhost:3000): http://localhost:3000
Insert profile name: base-profile

アカウントの資格情報がターミナルで確認できます。

Profile stored correctly
┌─────────────┬──────────────────────────────────────────────────────────────────┐
│ Property    │ Value                                                            │
├─────────────┼──────────────────────────────────────────────────────────────────┤
│ Address     │ SCVG35-ZSPMYP-L2POZQ-JGSVEG-RYOJ3V-BNIU3U-N2E6                   │
├─────────────┼──────────────────────────────────────────────────────────────────┤
│ Public Key  │ 654...321                                                        │
├─────────────┼──────────────────────────────────────────────────────────────────┤
│ Private Key │ 123...456                                                        │
└─────────────┴──────────────────────────────────────────────────────────────────┘

その名が示すように 秘密鍵は常に秘匿されていなければなりません 。秘密鍵にアクセスできる人はアカウントのコントロールを完全に掌握します。一方で、あなたのアカウントの公開鍵とアドレスはネットワーク上の他の参加者と安全に共有でき、彼らからのトランザクションを受け取ることができるようになります。

Next steps¶

Now that you have your test network running and an account with some cat.currency, we recommend you to surf the Symbol Developer Documentation. In this portal, you can find detailed information about Symbol’s features and self-paced guides on how to use the software development kits.
